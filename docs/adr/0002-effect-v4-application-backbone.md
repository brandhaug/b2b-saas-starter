# Effect v4 application backbone

Effect v4 is the application backbone across server, domain, infrastructure, and typed HTTP contracts. Capabilities, the API worker, the background worker, and the public `HttpApi` contract are all expressed in Effect. The React UI stays idiomatic React/TanStack for rendering: `apps/web` route loaders run capability Effects directly server-side (no REST round-trips to `apps/api`), and mutations go through TanStack Start server functions that wrap a capability call. `AtomHttpApi.Service`, `@effect/atom-react` query atoms, and `FetchHttpClient` with schema decoding remain the documented path for client-side capability querying over the HTTP API, but are not wired today — see [client server-state](./0040-effect-atom-for-server-state.md).
