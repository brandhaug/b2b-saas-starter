# Intent node documentation

The starter uses fresh `AGENTS.md` intent nodes modeled after Hexwardens' structure rather than copied from Hexwardens. The root node points to `CONTEXT.md`, `ARCHITECTURE.md`, and `DESIGN.md`, plus child intent nodes for the web app, API worker, background worker, database, and capabilities so agent guidance matches the starter domain instead of leaking game-specific invariants. The capabilities node maps the per-capability leaf nodes under its bounded-context folders. Smaller packages such as auth, email, and env do not yet carry their own intent node — their guidance lives in the consuming app nodes and package source until one grows enough to warrant a dedicated node.
