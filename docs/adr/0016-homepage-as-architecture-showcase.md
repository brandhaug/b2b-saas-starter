# Homepage as architecture showcase

The homepage should present the starter repository as an interactive architecture and capability showcase rather than a conventional generic SaaS landing page. It should use Contributor's visual polish and component patterns while making the first viewport clearly about inspecting a production starter stack: TanStack Start, Cloudflare Workers, Better Auth, Drizzle D1, Effect v4, Alchemy, REST/MCP, email, testing, and observability.
