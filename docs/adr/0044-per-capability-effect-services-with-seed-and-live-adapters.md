# Per-capability Effect services with Seed and Live adapters

The starter exposes each Starter Module-shaped Capability Interface as its own `Effect.Service`: Starter Module Catalog, Workspace Membership, Notification Feed, API Token Registry, Webhook Endpoints, Audit Event Log, Integration Surfaces, Implementation Reports, Catalog Refresh History, plus a pure Adoption Readiness projection. Each service ships two adapters: a deterministic in-memory Seed layer for local dev, Storybook, and tests, and a Drizzle-backed Live layer that depends on a `Database` service provided by `packages/db`. Workspace data shapes and the Seed Workspace fixture are internal to `packages/capabilities`; consumers compose `SeedLayer` or `makeLiveLayerFromD1(d1)` rather than importing the fixture directly. REST and MCP Capability Interfaces are derived from one HttpApi.Group per capability in `packages/api`, with `apps/api` dispatching through the same capability seam. This replaces the prior `WorkspaceOverview` god-object and the `@b2b-saas-starter/capabilities/seed` subpath import.
